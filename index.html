SISR · Cisco Packet Tracer — Contrôle Admin

Un seul routeur relie deux infrastructures partageant les mêmes VLANs. Infra 1 — Switch0 + PC0/PC1/PC2/PC3 sur GE0/0 en Router-on-a-Stick. Infra 2 — Switch1 + PC4/PC5 sur GE0/1, mêmes VLAN 100 et 200. Inter-VLAN désactivé — SISR et SLAM ne communiquent pas. DHCP géré par le routeur — les mêmes pools servent les deux infrastructures.

Topologie
  [Switch0]                      [Router0]                    [Switch1]
  /   |   \   \                  /       \                     /      \
[PC0][PC1][PC2][PC3]         GE0/0     GE0/1               [PC4]    [PC5]
VLAN100 VLAN100 VLAN200 VLAN200  |         |              VLAN100  VLAN200
                             (trunk)   (trunk)
                      GE0/0.10 → 192.168.10.1  (VLAN 100 — SISR)
                      GE0/0.20 → 192.168.20.1  (VLAN 200 — SLAM)
                      GE0/1.10 → 192.168.10.1  (VLAN 100 — SISR)
                      GE0/1.20 → 192.168.20.1  (VLAN 200 — SLAM)
● VLAN 100 SISR — 192.168.10.0/24 — passerelle 192.168.10.1 ● VLAN 200 SLAM — 192.168.20.0/24 — passerelle 192.168.20.1
1 Switch0

PC0 et PC1 → VLAN 100 (SISR) · PC2 et PC3 → VLAN 200 (SLAM) · Trunk vers Router0.

enable
configure terminal

vlan 100
 name SISR
exit
vlan 200
 name SLAM
exit

! PC0 et PC1 → VLAN 100
interface range fastEthernet 0/1 - 2
 switchport mode access
 switchport access vlan 100

! PC2 et PC3 → VLAN 200
interface range fastEthernet 0/3 - 4
 switchport mode access
 switchport access vlan 200

! Trunk vers Router0
interface gigabitEthernet 0/1
 switchport mode trunk

write memory
2 Router0 — GE0/0 + DHCP

Sous-interfaces GE0/0.10 et GE0/0.20 + pools DHCP SISR et SLAM. Ces pools serviront aussi l'Infra 2.

enable
configure terminal

! Sous-interface VLAN 100 — SISR

 no shutdown
exit

! Sous-interface VLAN 200 — SLAM

ip dhcp excluded-address 192.168.20.1

exit

! Pool VLAN 200 — SLAM

 default-router 192.168.20.1
 dns-server 8.8.8.8
exit

write memory

SISR · Cisco Packet Tracer

Deux réseaux reliés par Router1 qui assure le routage entre eux. Réseau gauche 192.168.10.0/24 — Router0 + Switch0 (PC0, PC1, Server0). Réseau droit 192.168.20.0/24 — Router1 + Switch1 (PC2, PC3, PC4, Server1). Les PC des deux réseaux peuvent se pinguer.

Topologie
  [Router0]——[Switch0]————[Router1]————[Switch1]
               /    \      /      \     /  |   \   \
           [PC0] [PC1] [GE0/0] [GE0/1][PC2][PC3][PC4][Server1]
             |
         [Server0]

   192.168.10.0/24   ←— routage —→   192.168.20.0/24
   GE0/0 → 10.1    Router1 : 10.2/20.1   GE0/1 → 20.1

4 DHCP — distribution automatique des IPs

Chaque routeur distribue les IPs de son réseau. Router0 gère le réseau gauche, Router1 le réseau droit.

! ── Router0 — DHCP réseau gauche ──
ip dhcp excluded-address 192.168.10.1
ip dhcp excluded-address 192.168.10.2

ip dhcp pool VLAN_GAUCHE
 network 192.168.10.0 255.255.255.0
 default-router 192.168.10.1
 dns-server 8.8.8.8
exit
write memory

! ── Router1 — DHCP réseau droit ──
ip dhcp excluded-address 192.168.20.1

ip dhcp pool VLAN_DROIT
 network 192.168.20.0 255.255.255.0
 default-router 192.168.20.1
 dns-server 8.8.8.8
exit
write memory
